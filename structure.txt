PROJECT: TAXI4U
Last updated: 2026-04-17

---

CURRENT STATE:
- FastAPI backend operational
- Zone fare matrix loaded from `fares.json` via absolute file path
- Zone polygon data loaded from `backend/data/zones.json` via absolute file path
- Geocoding via Nominatim, Canada-only, Cochrane-anchored for local inputs
- Routing via OSRM for live driving distance and duration
- Coordinate passthrough supported from mobile autocomplete selections
- Three-tier zone detection: coords -> geocoded -> raw
- React Native / Expo mobile app with autocomplete, fare display, and live zone map

---

BACKEND FILES:

  main.py
    FastAPI entry point.
    Endpoints: GET /, GET /zones, POST /fare/calculate, GET /fare/test.
    Handles input validation, coordinate passthrough, route lookup, zone detection,
    response assembly, and graceful fallback behavior.

  calculator.py
    Zone fare via fares.json matrix lookup + distance fare at $2/km.
    Loads fares.json using Path(__file__).resolve().parent / "fares.json".

  zone_mapper.py
    Keyword + coordinate-based zone detection.
    Loads polygon data from backend/data/zones.json at startup and derives
    boundary checks from the polygon data.

  geocoder.py
    Nominatim geocoding with Canada-only restriction and light input normalization.

  routing.py
    OSRM driving distance and duration between two coordinate points.

  fares.json
    Source of truth for pricing.

  backend/data/zones.json
    Source of truth for zone polygons, priorities, and colors.

---

MOBILE FILES (taxi4u-mobile/):

  App.js
    Entry point. Mounts AppNavigator.

  app.json
    Expo config.
    Foreground location enabled for the live map.
    Background location explicitly disabled:
    - isAndroidBackgroundLocationEnabled: false
    - isIosBackgroundLocationEnabled: false

  package.json
    Expo 54 app with react-navigation, react-native-maps, expo-location, expo-asset.

  src/config.js
    API_BASE. Update to the local machine IP before running on device.

  src/navigation/AppNavigator.js
    Stack flow: Home -> Result -> Map

  src/screens/HomeScreen.js
    Address input with direct-to-Nominatim autocomplete.
    Selected suggestions preserve lat/lon.
    Manual edits clear saved coords.

  src/screens/ResultScreen.js
    Fare result display and entry point to the live map.

  src/screens/MapScreen.js
    Fetches zones from backend /zones.
    Draws polygons and tracks live foreground driver location.

  src/services/api.js
    calculateFare()
    fetchZones()
    searchAddresses()

  src/services/backgroundLocation.js
    Stub only. Background tracking not implemented.

  src/data/zones.js
    Thin wrapper around fetchZones() with sorting and normalization.

  src/utils/zoneDetection.js
    Point-in-polygon detection on-device.

---

RUN BACKEND:
  cd TAXI4U
  venv\Scripts\activate
  python -m uvicorn main:app --reload --host 0.0.0.0 --port 8001

RUN MOBILE:
  cd TAXI4U/taxi4u-mobile
  npm install
  npx expo start

NOTE:
  Backend data loading does not depend on running from repo root.
  `fares.json` and `backend/data/zones.json` are both loaded via absolute
  file paths derived from the Python source files.
